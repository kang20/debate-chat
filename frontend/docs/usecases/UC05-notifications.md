# UC05: 알림

## 개요
토론 활동에 대한 알림을 수신하고 관리하는 기능을 제공한다.

---

## UC05-1: 알림 수신 (헤더 표시)

### 기본 흐름
1. 다른 사용자가 멘션, 논거 답글, 투표 등의 활동을 한다.
2. 헤더의 벨 아이콘 옆에 읽지 않은 알림 수가 빨간 배지로 표시된다.
3. 배지 숫자가 실시간으로 갱신된다 (9개 초과 시 "9+"로 표시).

### 알림 유형
| 유형 | 트리거 | 메시지 |
|------|--------|--------|
| MENTION | @닉네임으로 멘션 | "OOO님이 토론방에서 회원님을 멘션했습니다." |
| REPLY | 메시지에 답글 | "OOO님이 답글을 달았습니다." |
| NEW_MESSAGE | 참여 중인 토론방에 새 메시지 | "OOO님이 새 메시지를 보냈습니다." |
| ARGUMENT_REPLY | 내 논거에 답글 논거 | "OOO님이 논거에 답글을 달았습니다." |
| VOTE | 내 논거에 투표 | "OOO님이 논거에 투표했습니다." |
| ROOM_CLOSED | 참여 중인 토론방 종료 | "OOO님이 토론을 종료했습니다." |

### UI 매핑
- 화면: `Header` (벨 아이콘 + 배지)

---

## UC05-2: 알림 목록 조회

### 액터
- 주 액터: 인증된 사용자

### 기본 흐름
1. 헤더의 벨 아이콘을 클릭하거나 `/notifications` 페이지에 접근한다.
2. 알림 목록이 최신순으로 표시된다.
3. 읽지 않은 알림은 보라색 배경으로 강조된다.
4. 각 알림에는 유형 아이콘, 발신자, 내용 요약, 시간이 표시된다.
5. 알림을 클릭하면 해당 토론방으로 이동하고, 읽음 처리된다.

### UI 매핑
- 화면: `NotificationsPage`
- 관련 API: `GET /api/notifications?page=&unreadOnly=`

---

## UC05-3: 모두 읽음 처리

### 기본 흐름
1. 알림 페이지에서 "모두 읽음" 버튼을 클릭한다.
2. 모든 읽지 않은 알림이 읽음 상태로 변경된다.
3. 헤더의 배지가 사라진다.

### UI 매핑
- 화면: `NotificationsPage` (모두 읽음 버튼)
- 관련 API: `POST /api/notifications/read-all`

---

## UC05-4: 알림 설정 (향후 구현)

### 기본 흐름
1. 알림 설정 페이지에서 각 알림 유형별 on/off를 토글한다.
2. 비활성화된 유형의 알림은 더 이상 생성되지 않는다.

### 관련 API: `GET /api/notifications/preferences`, `PUT /api/notifications/preferences`
