# UC03: 실시간 채팅

## 개요
토론방 내에서 참여자들이 실시간으로 메시지를 교환하는 기능을 제공한다. 각 메시지에는 작성자의 입장(찬성/반대/중립) 배지가 표시된다.

---

## UC03-1: 메시지 전송

### 액터
- 주 액터: 토론방에 참여 중인 사용자

### 사전 조건
- 사용자가 토론방에 참여한 상태
- 토론방이 OPEN 상태

### 기본 흐름
1. 채팅 탭의 하단 입력 영역에 메시지를 입력한다.
2. Enter 키를 누르거나 전송 버튼을 클릭한다.
3. 메시지가 채팅 목록에 즉시 표시된다.
4. 메시지 옆에 작성자의 현재 입장 배지(PRO=파란/CON=빨간/NEUTRAL=회색)가 표시된다.
5. 자동으로 최하단으로 스크롤된다.

### 대안 흐름
- 빈 메시지인 경우: 전송 버튼이 비활성화 상태

### 사후 조건
- Message 엔티티가 생성됨
- sideAtSend에 작성 시점의 입장이 기록됨

### UI 매핑
- 화면: `ChatPanel`, `ChatInput`
- 관련 API: `POST /api/rooms/{roomId}/messages`

---

## UC03-2: 메시지 히스토리 로드

### 기본 흐름
1. 토론방 상세 페이지에 진입하면 최근 메시지가 자동 로드된다.
2. 채팅 영역 상단으로 스크롤하면 이전 메시지가 추가 로드된다 (커서 기반 페이지네이션).
3. 더 이상 이전 메시지가 없으면 "이전 메시지 불러오기" 버튼이 사라진다.

### UI 매핑
- 화면: `ChatPanel`
- 관련 API: `GET /api/rooms/{roomId}/messages?cursor=`

---

## UC03-3: 실시간 메시지 수신 (시뮬레이션)

### 기본 흐름
1. 토론방에 입장하면 채팅 시뮬레이터가 자동으로 시작된다.
2. 4~10초 간격으로 다른 참여자(봇)가 메시지를 보낸다.
3. 새 메시지가 실시간으로 채팅 목록에 추가된다.
4. 사용자가 하단에 위치하면 자동 스크롤, 상단을 보고 있으면 스크롤하지 않음.

### 참고
- 실제 백엔드 연동 시 WebSocket (STOMP) `/topic/rooms/{roomId}`를 구독

---

## UC03-4: 메시지 수정 (향후 구현)

### 비즈니스 규칙
- 본인 메시지만 수정 가능
- 작성 후 30분 이내에만 수정 가능
- 수정된 메시지에 "(수정됨)" 표시

### 관련 API: `PATCH /api/messages/{id}`

---

## UC03-5: 메시지 삭제 (향후 구현)

### 비즈니스 규칙
- 본인 메시지만 삭제 가능
- 소프트 삭제: "[삭제된 메시지]"로 표시

### 관련 API: `DELETE /api/messages/{id}`
