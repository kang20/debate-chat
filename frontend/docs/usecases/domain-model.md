# 도메인 모델 용어 사전

## 핵심 엔티티

| 한국어 | 영어 | 설명 |
|--------|------|------|
| 사용자 | User | 시스템에 가입한 회원. 이메일, 닉네임, 자기소개 등의 프로필 정보를 가진다. |
| 토론방 | DebateRoom | 하나의 토론 주제를 다루는 공간. 방장이 생성하며 공개/비공개, 진행중/종료 상태를 가진다. |
| 태그 | Tag | 토론방을 분류하기 위한 카테고리 (예: 기술, 정책, 경제). |
| 참여자 | RoomParticipant | 토론방에 입장한 사용자. 입장 시 찬성/반대/중립 중 하나를 선택한다. |
| 메시지 | Message | 토론방 내 실시간 채팅 메시지. 작성 시점의 입장(sideAtSend)이 기록된다. |
| 논거 | Argument | 구조화된 찬반 논리 트리의 노드. 부모-자식 관계로 반박/보충 구조를 형성한다. |
| 투표 | ArgumentVote | 논거에 대한 동의(AGREE) 또는 비동의(DISAGREE) 표현. |
| 알림 | Notification | 사용자에게 전달되는 활동 알림 (멘션, 답글, 투표 등). |

## 열거형 (Enum)

| 이름 | 값 | 설명 |
|------|-----|------|
| Side (입장) | `PRO` / `CON` / `NEUTRAL` | 찬성 / 반대 / 중립 |
| Visibility (공개여부) | `PUBLIC` / `PRIVATE` | 공개 / 비공개 |
| RoomStatus (방 상태) | `OPEN` / `CLOSED` | 진행중 / 종료 |
| VoteValue (투표값) | `AGREE` / `DISAGREE` | 동의 / 비동의 |
| NotificationType (알림유형) | `MENTION` / `REPLY` / `NEW_MESSAGE` / `ARGUMENT_REPLY` / `VOTE` / `ROOM_CLOSED` | 알림 종류 |

## 엔티티 관계도

```
User ──< RoomParticipant >── DebateRoom ──< Tag
  │                                │
  │                                ├──< Message
  │                                │
  │                                └──< Argument ──< ArgumentVote
  │                                       │
  └──< Notification                       └──< Argument (자식 논거)
```

## 핵심 비즈니스 규칙

1. **입장 선택**: 토론방 참여 시 반드시 PRO/CON/NEUTRAL 중 하나를 선택해야 한다.
2. **메시지 입장 스냅샷**: 메시지 전송 시 작성자의 현재 입장이 기록된다 (sideAtSend).
3. **메시지 수정 제한**: 본인 메시지만 수정 가능하며, 작성 후 30분 이내에만 가능하다.
4. **소프트 삭제**: 메시지와 논거는 물리적으로 삭제되지 않고 deleted 플래그로 처리된다.
5. **논거 트리**: 부모 논거에 대한 반박(CON) 또는 보충(PRO) 논거를 자식으로 추가할 수 있다.
6. **투표 제한**: 한 사용자는 하나의 논거에 대해 하나의 투표만 가능하다 (변경/취소 가능).
7. **토론 종료**: 방장만 토론을 종료할 수 있으며, 종료 후 새 메시지/논거 추가 불가.
