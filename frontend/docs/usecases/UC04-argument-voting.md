# UC04: 논거 트리 및 투표

## 개요
토론의 논점을 구조화된 트리 형태로 정리하고, 각 논거에 동의/비동의 투표를 할 수 있는 기능을 제공한다.

---

## UC04-1: 논거 추가

### 액터
- 주 액터: 토론방에 참여 중인 사용자

### 사전 조건
- 사용자가 토론방에 참여한 상태
- 토론방이 OPEN 상태

### 기본 흐름 (루트 논거)
1. 논거 트리 탭에서 "논거 추가" 버튼을 클릭한다.
2. 논거 추가 폼이 표시된다.
3. 입장(찬성/반대)을 선택한다.
4. 논거 내용을 입력한다.
5. "추가" 버튼을 클릭한다.
6. 논거가 트리의 루트 레벨에 추가된다.

### 기본 흐름 (답글 논거)
1. 기존 논거의 "답글" 버튼을 클릭한다.
2. 해당 논거 하위에 인라인 폼이 표시된다.
3. 입장(찬성/반대)을 선택하고 내용을 입력한다.
4. "추가" 버튼을 클릭한다.
5. 부모 논거의 자식으로 추가된다.

### 시각적 표현
- 찬성(PRO) 논거: 왼쪽에 파란색 보더
- 반대(CON) 논거: 왼쪽에 빨간색 보더
- 깊이(depth)에 따라 들여쓰기 적용

### 사후 조건
- Argument 엔티티가 생성됨 (parentId 설정)
- 트리가 자동 갱신됨

### UI 매핑
- 화면: `ArgumentTree`, `ArgumentForm`, `ArgumentNode`
- 관련 API: `POST /api/rooms/{roomId}/arguments`

## UC04-2: 논거 트리 탐색

### 기본 흐름
1. 논거 트리 탭을 선택하면 전체 트리가 로드된다.
2. 자식 논거가 있는 노드는 펼치기/접기 토글이 표시된다.
3. 토글을 클릭하면 하위 논거가 펼쳐지거나 접힌다.
4. 각 노드에는 작성자, 입장 배지, 작성 시간, 투표 수가 표시된다.

### UI 매핑
- 화면: `ArgumentTree`, `ArgumentNode` (재귀 컴포넌트)
- 관련 API: `GET /api/rooms/{roomId}/arguments`

---

## UC04-3: 투표 (동의/비동의)

### 액터
- 주 액터: 인증된 사용자

### 기본 흐름
1. 논거 노드의 👍(동의) 또는 👎(비동의) 버튼을 클릭한다.
2. 투표 수가 즉시 갱신된다.
3. 클릭한 버튼이 활성 상태로 표시된다.

### 대안 흐름
- 이미 같은 값으로 투표한 경우: 투표가 취소됨
- 다른 값으로 투표한 경우: 기존 투표가 변경됨

### 비즈니스 규칙
- 한 사용자는 하나의 논거에 하나의 투표만 가능
- 투표 변경 및 취소 가능
- 본인 논거에도 투표 가능

### UI 매핑
- 화면: `ArgumentNode` (VoteButton 영역)
- 관련 API: `POST /api/arguments/{id}/votes`, `DELETE /api/arguments/{id}/votes/me`

## UC04-4: 논거 삭제

### 기본 흐름
1. 본인이 작성한 논거의 삭제 버튼을 클릭한다.
2. 논거가 소프트 삭제되어 "[삭제된 논거]"로 표시된다.
3. 하위 논거는 보존된다 (트리 구조 유지).

### 관련 API: `DELETE /api/arguments/{id}`
