# UC01: 인증 및 사용자 프로필

## 개요
사용자의 회원가입, 로그인, 로그아웃, 프로필 조회 및 수정 기능을 제공한다.

---

## UC01-1: 회원가입

### 액터
- 주 액터: 비인증 사용자

### 사전 조건
- 사용자가 아직 계정을 보유하지 않은 상태

### 기본 흐름
1. 사용자가 회원가입 페이지(`/auth/signup`)에 접근한다.
2. 이메일, 닉네임(2~20자), 비밀번호(8자 이상)를 입력한다.
3. "회원가입" 버튼을 클릭한다.
4. 시스템이 입력값을 검증한다.
5. 계정이 생성되고, 로그인 페이지로 리다이렉트된다.

### 대안 흐름
- 이미 사용 중인 이메일인 경우: 에러 메시지 표시 ("이미 사용 중인 이메일입니다.")
- 유효성 검증 실패: 해당 필드에 에러 메시지 표시

### 사후 조건
- 새로운 User 엔티티가 생성됨

### UI 매핑
- 화면: `SignupPage`
- 관련 API: `POST /api/auth/signup`

---

## UC01-2: 로그인

### 액터
- 주 액터: 가입된 사용자

### 사전 조건
- 유효한 계정이 존재

### 기본 흐름
1. 사용자가 로그인 페이지(`/auth/login`)에 접근한다.
2. 이메일과 비밀번호를 입력한다.
3. "로그인" 버튼을 클릭한다.
4. 시스템이 자격 증명을 검증하고 JWT 토큰을 발급한다.
5. 토큰이 localStorage에 저장되고, 홈 페이지로 리다이렉트된다.
6. 헤더에 사용자 닉네임과 아바타가 표시된다.

### 대안 흐름
- 잘못된 자격 증명: 에러 토스트 메시지 표시

### 사후 조건
- 인증 토큰이 저장됨
- authStore에 사용자 정보가 로드됨

### UI 매핑
- 화면: `LoginPage`
- 관련 API: `POST /api/auth/login`, `GET /api/users/me`

---

## UC01-3: 로그아웃

### 기본 흐름
1. 사용자가 헤더의 로그아웃 버튼을 클릭한다.
2. localStorage에서 토큰이 삭제된다.
3. authStore 상태가 초기화된다.
4. 홈 페이지로 리다이렉트된다.

### 관련 API: `POST /api/auth/logout`

---

## UC01-4: 프로필 조회/수정

### 기본 흐름
1. 사용자가 프로필 페이지(`/profile`)에 접근한다.
2. 현재 닉네임, 이메일, 자기소개, 가입일이 표시된다.
3. "프로필 수정" 버튼을 클릭하면 편집 모드로 전환된다.
4. 닉네임, 자기소개를 수정하고 "저장" 버튼을 클릭한다.
5. 수정 사항이 저장되고 뷰 모드로 돌아간다.

### UI 매핑
- 화면: `ProfilePage`
- 관련 API: `GET /api/users/me`, `PATCH /api/users/me`
