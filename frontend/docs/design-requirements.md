# 토론 채팅 서비스 — 비즈니스 요구사항

## 서비스 개요

사용자들이 다양한 사회적 주제에 대해 **찬성/반대/중립** 입장을 정하고, 실시간 채팅과 구조화된 논거 트리를 통해 토론하며, 투표로 설득력을 평가하는 서비스입니다.

---

## 1. 사용자 및 인증

### 1.1 회원가입
- 이메일, 닉네임(2~20자), 비밀번호(8자 이상)를 입력하여 가입한다.
- 이메일과 닉네임은 중복 불가.
- 가입 완료 시 로그인 화면으로 이동한다.

### 1.2 로그인 / 로그아웃
- 이메일과 비밀번호로 로그인한다.
- 로그인 후 헤더에 사용자 닉네임과 아바타가 표시된다.
- 로그아웃 시 초기 화면으로 돌아간다.

### 1.3 프로필
- 닉네임, 이메일(읽기 전용), 자기소개, 가입일을 확인할 수 있다.
- 닉네임과 자기소개는 수정 가능하다.

---

## 2. 토론방

### 2.1 토론방 생성
- 로그인한 사용자가 새 토론방을 만들 수 있다.
- 입력 항목: **토론 주제**(필수), 설명, 태그(기술/정치/경제/사회/윤리 등).
- 생성자는 자동으로 "중립" 입장으로 참여된다.
- 생성 후 해당 토론방으로 이동한다.

### 2.2 토론방 목록 / 검색
- 메인 화면에 토론방이 카드 형태로 나열된다.
- 각 카드에 표시될 정보:
  - 토론 주제 (제목)
  - 상태 (진행중 / 종료)
  - 태그
  - 참여자 수 (찬성 · 반대 · 중립 각각)
- 검색: 키워드로 제목/설명 검색
- 필터: 상태별(전체/진행중/종료), 태그별
- 페이지네이션 지원

### 2.3 토론방 참여
- 토론방에 들어가면 **찬성(PRO) / 반대(CON) / 중립(NEUTRAL)** 중 입장을 선택해야 참여할 수 있다.
- 참여하지 않은 사용자는 채팅과 논거를 **읽기만** 가능하다.
- 참여자는 언제든 "나가기"로 퇴장할 수 있다.

### 2.4 토론방 종료
- 방장(생성자)만 토론을 종료할 수 있다.
- 종료되면 더 이상 새 메시지/논거 추가가 불가능하다.
- 기존 내용은 열람 가능.

---

## 3. 실시간 채팅

### 3.1 메시지 전송
- 참여 중인 사용자만 메시지를 보낼 수 있다.
- 각 메시지에는 작성자의 **현재 입장 배지**(찬성=파란색, 반대=빨간색, 중립=회색)가 함께 표시된다.
- 전송 시 자동으로 최하단 스크롤.

### 3.2 메시지 히스토리
- 토론방 진입 시 최근 메시지가 자동 로드된다.
- 위로 스크롤하면 이전 메시지가 추가로 로드된다.

### 3.3 메시지 수정/삭제 규칙
- 본인 메시지만 수정/삭제 가능.
- 수정은 **작성 후 30분 이내**에만 가능하며, "(수정됨)" 표시가 붙는다.
- 삭제 시 "삭제된 메시지"로 표시되고 원본은 복구 불가.

---

## 4. 논거 트리 및 투표

### 4.1 논거(Argument)란?
채팅이 자유로운 대화 흐름이라면, 논거는 **구조화된 찬반 논리**입니다.
트리 형태로 표현되며, 어떤 논거에 대한 반박이나 보충이 자식 노드로 달립니다.

### 4.2 논거 추가
- 참여 중인 사용자만 추가 가능.
- **루트 논거**: 토론의 핵심 주장을 새로 등록.
- **답글 논거**: 기존 논거에 대한 반박(반대) 또는 보충(찬성)을 자식으로 등록.
- 각 논거에 **찬성(PRO) 또는 반대(CON)** 입장을 선택해야 한다.

### 4.3 논거 트리 시각화
- 찬성 논거: 왼쪽에 **파란색 보더**
- 반대 논거: 왼쪽에 **빨간색 보더**
- 깊이(depth)에 따라 **들여쓰기** 적용
- 자식이 있는 노드는 **펼치기/접기** 토글 가능
- 각 노드에 표시: 작성자 닉네임, 입장 배지, 내용, 작성 시간, 동의/비동의 수

### 4.4 투표
- 로그인한 사용자는 각 논거에 **동의(👍)** 또는 **비동의(👎)** 투표를 할 수 있다.
- 한 논거당 한 표만 가능 (변경·취소 가능).
- 같은 버튼을 다시 누르면 투표 취소, 반대 버튼을 누르면 투표 변경.
- 투표 수는 실시간으로 반영된다.

### 4.5 논거 삭제
- 본인 논거만 삭제 가능.
- 삭제 시 "삭제된 논거"로 표시되고, **하위 논거는 보존**된다 (트리 구조 유지).

## 5. 알림

### 5.1 알림 유형

| 상황 | 알림 메시지 |
|------|------------|
| 누군가 나를 @멘션 | "OOO님이 토론방에서 회원님을 멘션했습니다." |
| 내 메시지에 답글 | "OOO님이 답글을 달았습니다." |
| 참여 중인 방에 새 메시지 | "OOO님이 새 메시지를 보냈습니다." |
| 내 논거에 답글 논거 | "OOO님이 논거에 답글을 달았습니다." |
| 내 논거에 투표 | "OOO님이 논거에 투표했습니다." |
| 참여 중인 토론 종료 | "OOO님이 토론을 종료했습니다." |

### 5.2 알림 표시
- 헤더의 벨 아이콘 옆에 읽지 않은 알림 수가 빨간 배지로 표시된다 (9개 초과 시 "9+").
- 알림 페이지에서 전체 목록을 확인할 수 있다.
- 읽지 않은 알림은 강조 표시된다.
- 알림 클릭 시 해당 토론방으로 이동하며 자동으로 읽음 처리된다.
- "모두 읽음" 버튼으로 일괄 처리 가능.

| 입장 | 한국어 | 색상 | 사용 위치 |
|------|--------|------|-----------|
| PRO | 찬성 | 파란색 | 메시지 배지, 논거 보더, 참여자 목록, 입장 선택 |
| CON | 반대 | 빨간색 | 메시지 배지, 논거 보더, 참여자 목록, 입장 선택 |
| NEUTRAL | 중립 | 회색 | 메시지 배지, 참여자 목록, 입장 선택 |

## 8. 화면 구성

### 8.1 메인 화면 (토론방 목록)
- 상단: 검색바 + 필터 (상태, 태그)
- 중앙: 토론방 카드 그리드
- 하단: 페이지네이션
- 로그인 시 "새 토론" 버튼 표시

### 8.2 토론방 상세
- 상단: 토론 주제, 설명, 태그, 방장 정보, 참여/나가기/종료 버튼
- 탭 전환: **채팅** | **논거 트리**
- 채팅 탭: 메시지 목록 + 하단 입력창
- 논거 트리 탭: 트리 시각화 + 논거 추가 폼
- 우측 사이드바: 참여자 목록 (입장별 그룹핑: 찬성 N명, 반대 N명, 중립 N명)

### 8.3 인증 화면
- 로그인: 이메일 + 비밀번호 입력 폼
- 회원가입: 이메일 + 닉네임 + 비밀번호 입력 폼
- 하단에 로그인↔회원가입 전환 링크

### 8.4 프로필 화면
- 아바타 + 닉네임 + 이메일 + 자기소개 + 가입일
- 편집 모드 전환 가능

### 8.5 알림 화면
- 상단: "모두 읽음" 버튼
- 알림 카드 리스트: 유형 아이콘 + 발신자 + 내용 요약 + 시간 + 읽음 상태

## 9. 핵심 비즈니스 규칙 요약

1. 토론방 참여 시 반드시 **찬성/반대/중립** 중 입장을 선택해야 한다.
2. 메시지에는 작성 시점의 입장이 기록되어 배지로 표시된다.
3. 메시지 수정은 **30분 이내**, 본인만 가능.
4. 논거는 부모-자식 트리 구조이며, 삭제해도 하위 논거는 보존된다.
5. 투표는 논거당 **1인 1표** (변경·취소 가능).
6. 토론 종료는 **방장만** 가능하며, 종료 후 읽기 전용이 된다.
7. 삭제된 메시지/논거는 물리 삭제가 아닌 "삭제됨" 표시로 처리한다.

## 10. 예시 토론 주제

실제 서비스에서 다루는 토론 주제 예시:
- "AI 규제, 지금 당장 필요한가?"
- "원격근무 vs 출근, 어떤 근무 방식이 더 생산적인가?"
- "기본소득제 도입, 찬성인가 반대인가?"
- "소셜미디어가 민주주의에 미치는 영향은?"
