# 토론 채팅 서비스 — 비즈니스 요구사항

## 서비스 개요

사용자들이 다양한 사회적 주제에 대해 **찬성/반대/중립** 입장을 정하고, 실시간 채팅과 구조화된 논거 트리를 통해 토론하며, 투표로 설득력을 평가하는 서비스입니다.

---

## 1. 사용자 및 인증

### 1.1 회원가입
- OAuth(구글, 카카오)로 회원가입 가능하다.
  - 계정당 중복 회원가입 불가.
- 계정당 닉네임 정보를 추가로 입력 받는다. (필수)
  - 닉네임은 2~20자 사이의 한글, 영어, 숫자 조합으로 허용한다.
  - 닉네임은 고유해야 하며, 중복 시 가입 불가 메시지를 표시한다.
- 가입 완료 시 로그인 화면으로 이동한다.

### 1.2 로그인 / 로그아웃
- OAuth(구글, 카카오)로 로그인 가능하다.
- 로그인 후 헤더에 사용자 닉네임과 아바타가 표시된다.
- 로그아웃 시 초기 화면으로 돌아간다.

### 1.3 프로필
- 닉네임, 연동 계정, 가입일을 확인할 수 있다.
- 사용자의 토론 등급을 확인할 수 있다.

## 2. 토론방

### 2.1 토론방 생성
- 관리자 권한 또는 특정 등급 이상의 로그인한 사용자만 토론방을 생성할 수 있다.
- 입력 항목:
  - **토론 주제** (필수)
  - **설명** (필수)
  - **태그**: 직접 입력(Enter로 추가) 또는 자주 쓰는 태그 클릭으로 추가. X 버튼으로 제거 가능.
  - **참여 인원** (찬반 합친 최소/최대 인원수)
    - 최소 인원수는 최소 2명 이상이어야 한다.
  - **시작 일시**
- 자주 쓰는 태그: 기술, 정책, 경제, 사회, 직장, 복지, 라이프스타일
- 생성자는 자동으로 **진행자** 역할로 참여된다.
- 생성 후 해당 토론방으로 이동한다.

### 2.2 토론방 상태

| 상태 | 조건                                                       |
|------|----------------------------------------------------------|
| 모집 중 | 생성 직후. 시작 시간 전까지 참여자를 모집한다. (단 최소 인원수 미만이면 종료로 바로 변경된다.) |
| 진행 중 | 시작 시간이 되면 자동 전환, 또는 진행자가 수동으로 "다음 단계로" 전환.               |
| 종료 | 최종 변론 후 진행자가 종료. 읽기 전용.                                  |

### 2.3 토론방 목록
- 메인 화면에 토론방이 카드 형태로 나열된다.
- 각 카드에 표시될 정보: 토론 주제, 상태(모집 중/진행 중/종료), 태그, 참여자 수(찬성·반대·중립 각각), 시작 시간
- 정렬: 인기(참여자 수) / 최신(생성일) / 박빙(찬반 차이)

### 2.4 토론방 참여
- 토론방에 들어가면 **찬성(PRO) / 반대(CON) / 중립(NEUTRAL)** 중 입장을 선택해야 참여할 수 있다.
- 찬반 참여 인원이 상한에 도달하면 더 이상 찬성/반대로 참여할 수 없다. (중립은 제한 없음)
- 참여자는 언제든 "나가기"로 퇴장할 수 있다.

### 2.5 채팅방 구분
- 하나의 토론방에는 **토론 채팅**과 **중립 채팅** 두 개의 채팅 공간이 존재한다.
- **토론 채팅**: 찬반 참여자가 토론을 진행하는 공간.
  - 찬반 참여자만 메시지를 보낼 수 있다.
  - 찬반 참여자는 중립 채팅을 볼 수 없다.
- **중립 채팅**: 중립 참여자가 자유롭게 의견을 나누는 공간.
  - 중립 참여자만 메시지를 보낼 수 있다.
  - 중립 참여자는 토론 채팅과 중립 채팅을 모두 볼 수 있다.

### 2.6 토론방 종료
- **진행자**만 토론을 종료할 수 있다.
- 종료되면 더 이상 새 메시지/논거 추가가 불가능하다.
- 기존 내용은 열람 가능.

## 3. 토론 진행 단계

토론방은 **4단계**를 순서대로 거치며, 시스템이 각 단계의 목적과 행동 가이드를 안내한다.

### 3.1 단계 흐름

```
모집 → 입론 → 반론 → 최종 변론 → (종료)
```

| 단계 | 영어 | 목적 |
|------|------|------|
| 모집 | RECRUITING | 참여자를 모으고 찬성/반대 팀을 구성한다 |
| 입론 | OPENING | 각 팀이 핵심 주장과 근거를 발표한다 |
| 반론 | REBUTTAL | 상대 팀의 논리적 허점을 공격하고 질문을 통해 방어한다 |
| 최종 변론 | CLOSING | 자신의 주장을 정리하고 상대의 반론을 재반박하며 마무리한다 |

### 3.2 단계 전환
- **진행자만** 다음 단계로 진행할 수 있다 ("다음 단계로" 버튼).
- 모집 → 입론 전환은 시작 시간이 되면 **자동 전환**되거나, 진행자가 시작 시간 전에 수동 전환할 수 있다.
- 단계는 반드시 순서대로 진행되며, 건너뛰거나 되돌릴 수 없다.
- 최종 변론 이후 진행자가 "토론 종료"를 하면 토론이 마감된다.

### 3.3 시스템 가이드 메시지

단계가 전환될 때마다 시스템이 채팅에 **안내 메시지**를 자동 발송한다.

| 단계 전환 | 시스템 메시지 |
|-----------|-------------|
| 방 생성 시 | "토론이 생성되었습니다. 참여자를 모집 중입니다. 찬성 또는 반대 입장을 선택해 참여해주세요." |
| → 입론 | "입론 단계가 시작되었습니다. 각 팀은 핵심 주장과 근거를 발표해주세요. 논거 트리에 루트 논거를 등록하는 것을 권장합니다." |
| → 반론 | "반론 단계입니다. 상대 팀의 주장에 반박하고 질문해주세요. 상대 논거에 답글 논거를 달아보세요." |
| → 최종 변론 | "최종 변론입니다. 지금까지의 논의를 정리하고 마무리 발언을 해주세요." |
| → 종료 | "토론이 종료되었습니다. 더 이상 새 메시지나 논거를 추가할 수 없습니다. 투표 결과를 확인해보세요." |

### 3.4 단계별 허용 행동

| 행동 | 모집 | 입론 | 반론 | 최종 변론 | 종료 |
|------|:----:|:----:|:----:|:---------:|:----:|
| 참여/퇴장 | O | O | O | O | X |
| 토론 채팅 메시지 전송 | X | O | O | O | X |
| 중립 채팅 메시지 전송 | O | O | O | O | X |
| 루트 논거 등록 | X | O | O | O | X |
| 답글 논거 등록 | X | X | O | O | X |
| 논거 투표 | X | X | O | O | X |
| 다음 단계 전환 (진행자) | O | O | O | O | - |

### 3.5 단계 표시
- 토론방 상세 화면 상단에 **현재 단계 인디케이터**가 표시된다.
- 4단계를 스텝 바(step bar) 형태로 시각화하며, 현재 단계가 강조된다.
- 각 단계 옆에 간략한 설명 툴팁을 제공한다.

## 4. 실시간 채팅

### 4.1 채팅 공간
- 토론방에는 **토론 채팅**과 **중립 채팅** 두 개의 탭이 존재한다 (2.5 채팅방 구분 참조).
- 각 채팅 공간은 독립적인 메시지 스트림을 가진다.

### 4.2 메시지 전송
- 해당 채팅 공간에 권한이 있는 참여자만 메시지를 보낼 수 있다.
- 각 메시지에는 작성자의 **현재 입장 배지**(찬성=파란색, 반대=빨간색, 중립=회색)가 함께 표시된다.
- 전송 시 자동으로 최하단 스크롤.

### 4.3 메시지 히스토리
- 토론방 진입 시 최근 메시지가 자동 로드된다.
- 위로 스크롤하면 이전 메시지가 추가로 로드된다.

### 4.4 메시지 수정/삭제 규칙
- 본인 메시지만 수정/삭제 가능.
- 수정은 **작성 후 30분 이내**에만 가능하며, "(수정됨)" 표시가 붙는다.
- 삭제 시 "삭제된 메시지"로 표시되고 원본은 복구 불가.

## 5. 논거 트리 및 투표

### 5.1 논거(Argument)
- 채팅이 자유로운 대화 흐름이라면, 논거는 **구조화된 찬반 논리**이다.
- 트리 형태로 표현되며, 어떤 논거에 대한 반박이나 보충이 자식 노드로 달린다.

### 5.2 논거 추가
- 찬반 참여자만 추가 가능.
- **루트 논거**: 토론의 핵심 주장을 새로 등록. (입론 단계부터 가능)
- **답글 논거**: 기존 논거에 대한 반박(반대) 또는 보충(찬성)을 자식으로 등록. (반론 단계부터 가능)
- 각 논거에 **찬성(PRO) 또는 반대(CON)** 입장을 선택해야 한다.

### 5.3 논거 트리 시각화
- 찬성 논거: 왼쪽에 **파란색 보더**
- 반대 논거: 왼쪽에 **빨간색 보더**
- 깊이(depth)에 따라 **들여쓰기** 적용
- 자식이 있는 노드는 **펼치기/접기** 토글 가능
- 각 노드에 표시: 작성자 닉네임, 입장 배지, 내용, 작성 시간, 동의/비동의 수

### 5.4 투표
- 로그인한 사용자는 각 논거에 **동의** 또는 **비동의** 투표를 할 수 있다. (반론 단계부터 가능)
- 한 논거당 한 표만 가능 (변경·취소 가능).
- 같은 버튼을 다시 누르면 투표 취소, 반대 버튼을 누르면 투표 변경.
- 투표 수는 실시간으로 반영된다.

### 5.5 논거 삭제
- 본인 논거만 삭제 가능.
- 삭제 시 "삭제된 논거"로 표시되고, **하위 논거는 보존**된다 (트리 구조 유지).

## 7. 핵심 비즈니스 규칙 요약

1. 토론방 참여 시 반드시 **찬성/반대/중립** 중 입장을 선택해야 한다.
2. 토론방 생성자는 **진행자**가 되어 단계 전환과 토론 종료를 관리한다.
3. 토론은 **모집 → 입론 → 반론 → 최종 변론 → 종료** 순서로 진행된다.
4. 모집 → 입론 전환은 시작 시간에 자동 전환되거나, 진행자가 수동 전환할 수 있다.
5. 각 단계에서 허용되는 행동이 다르다 (모집: 논거 불가 / 입론: 답글 논거 불가 / 반론부터: 모두 가능).
6. 단계 전환 시 시스템이 채팅에 가이드 메시지를 자동 발송한다.
7. 채팅은 **토론 채팅**(찬반 전용)과 **중립 채팅**(중립 전용)으로 분리된다. 중립은 양쪽 모두 열람 가능.
8. 찬반 참여 인원은 상한이 있으며, 중립은 제한 없다.
9. 메시지에는 작성 시점의 입장이 기록되어 배지로 표시된다.
10. 메시지 수정은 **30분 이내**, 본인만 가능.
11. 논거는 부모-자식 트리 구조이며, 삭제해도 하위 논거는 보존된다.
12. 투표는 논거당 **1인 1표** (변경·취소 가능).
13. 삭제된 메시지/논거는 물리 삭제가 아닌 "삭제됨" 표시로 처리한다.
